# Mighty Mouse

Use the `mighty-mouse` MCP server when the user invokes `/mighty`, mentions the Mighty Mouse protocol, or requests a reliability-sensitive multi-file change.

- Call `protocol` before editing and select complexity from the actual blast radius.
- Follow the returned protocol without exposing private chain-of-thought; communicate concise evidence and decisions.
- If workspace identity is not configured, call `setup_workspace` once with the active local model and host profile; then call `verify_and_record` after editing with explicit commands when auto-detection is incomplete.
- Use `allowed_paths` for bounded tasks.
- Fix and re-run failed checks for at most three verification rounds.
- Distinguish passed, failed, skipped, and unverified checks.
